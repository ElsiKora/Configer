# Error Codes

Configer throws `ConfigError` with stable `CODE` values for runtime failures.

## Parse and Loader Errors

- `CONFIG_ENV_PARSE_ERROR`
- `CONFIG_JSON_PARSE_ERROR`
- `CONFIG_JSON5_PARSE_ERROR`
- `CONFIG_JSONC_PARSE_ERROR`
- `CONFIG_TOML_PARSE_ERROR`
- `CONFIG_YAML_PARSE_ERROR`
- `CONFIG_JS_IMPORT_ERROR`
- `CONFIG_JS_REQUIRE_ERROR`
- `CONFIG_PACKAGE_PARSE_ERROR`
- `CONFIG_LOADER_NOT_REGISTERED`

## Inheritance and Environment Errors

- `CONFIG_INHERITANCE_CYCLE`
- `CONFIG_INVALID_INHERITANCE_VALUE`
- `CONFIG_INVALID_INHERITANCE_ENTRY`
- `CONFIG_INVALID_ENVIRONMENT_OVERRIDE`
- `CONFIG_INVALID_ENVIRONMENT_MAP`
- `CONFIG_INVALID_ENVIRONMENT_MAP_ENTRY`
- `CONFIG_WORKSPACE_MANIFEST_PARSE_ERROR`

## Schema Errors

- `CONFIG_SCHEMA_VALIDATION_ERROR`

## Sync Mode Errors

- `CONFIG_SYNC_LOADER_MISSING`
- `CONFIG_SYNC_UNSUPPORTED_EXTENSION`
- `CONFIG_SYNC_FUNCTION_RETURNED_PROMISE`
- `CONFIG_SYNC_PLUGIN_HOOK_PROMISE`
- `CONFIG_SYNC_TRANSFORM_RETURNED_PROMISE`

## Debugging Tip

When handling errors, use `error.CODE` as machine-readable branch condition and `error.message`
for user-facing diagnostics.
